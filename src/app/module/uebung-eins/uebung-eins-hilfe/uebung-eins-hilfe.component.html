Übung - 1 - Hilfestellung

Hilfe - Syntax
Erklärung

Sicherlich hat dich die vorgegeben Funktion erst einmal vor ein Rätsel gestellt:

{{'export function uebungEins(faktor1, faktor2, eingabeProdukt)  { return (1*2) == 1 ;}'}}

Programmierung ist das Schreiben von Text. Jedoch muss der Text für den Computer verstehbar sein. Text ist nur dann für Computer verstehtbar, wenn er sehr strengen Regeln folgt. Dies wird durch eine ganz bestimmte Anordnung von bestimmten Schlüsselwörtern und-zeichen erreicht. Das ist sehr unintuitiv, aber lass Dich davon nicht abschrecken - es lohnt sich ;)

Lass uns einmal diese Syntax anhand unseres Beispiels durchgehen.

Schlüsselwörter
export

export ist ein Schlüsselwort um anzugeben, dass die Funktion nicht nur innerhalb dieser Datei verwendet werden kann. Zu dem Thema der Sichtbarkeit kommen wir jedoch später noch.

function

function ist das Wort mit dem eine Funktion beginnt. Funktionen kennen wir aus der Schulmathematik: f(x) = x+1.

In der Programmiersprache würde das function f(x) {{ '{ return x+1 }' }} heissen.

Eine function ist eine Reihe von Anweisungen mit keinem oder mehreren Eingabeparametern (wie 'x') und keinem oder genau einem Rückgabewert (siehe return)

function uebungEins(..)

In der Schule sah fast jede Funktion so aus: f(x) = x+1. Aber manche auch so h(x)=..

f, p, a, b oder sonst wie waren nur Namen. Und genauso ist es hier: uebungEins ist nur der Name.

Aber Namen sind wichtig, schließlich ist das alles hier schon komplex genug, deswegen sollten wir gute eindeutige Namen verwenden, niemals f,p oder l.

function uebungEins(faktor1, faktor2, eingabeProdukt)

Die Parameter faktor1, faktor2, eingabeProdukt sind Eingaben in die Funktion hinein.

Die Funktion berechnet also irgendetwas was von diesen drei Werten abhängt. Prinzipiell können keine bis unendlich viele Werte und nicht nur Zahlen, sondern auch Wörter oder Objekte übergeben werden.

function uebungEins(..) {{'{ .. }'}}

Die Geschweiften Klammern haben sich etabliert um einen Block zu kennzeichnen.

So wie die mathematischen runden Klammern funktionieren, so ähnlich sind die Geschweiften. Was zwischen {{'{ und }'}} passiert, bleibt auch dort ;).

return ..

So wie die Parameter die Eingabe der Funktion darstellen, so stellt das was hinter return kommt die Ausgabe dar.

Wird die Anweisung return erreicht, wird der Block (s.o.) verlassen. In diesem Fall ist das unsere Funktion, welche den dahinter stehenden Wert zurückgibt; also (1*2) == 1

(1*2) == 1

Der Ausdruck (1*2) == 1 ist ein mathematischer Ausdruck, welcher vom Computer einfach ausgewertet wird.

Zuerst wird (1*2) ausgewertet. Dann wird geschaut ob 2 == 1ist. Der Rückgabewert ist damit "wahr" oder "falsch".

Im Gegensatz zur Schulmathematik gibt es einen Unterschied zwischen = und ==. Bei = macht man eine Zuweisung, während == ein Vergleichsoperator, wie beispielsweise > oder < ist.

Semicolon ;

Das Semicolon ist das was in normaler Sprache ein Punkt oder ein Komma ist. Es ist dazu da einen "Satz" zu beenden, nur dass das was ein Satz ist in Programmiersprache eine Anweisung ist, wie bspw.: holMirEinenKaffe();
